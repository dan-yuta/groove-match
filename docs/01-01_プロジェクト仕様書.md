# BandMatch - 初心者バンドマン向けコミュニティアプリ 仕様書

## 概要

**BandMatch** は、初心者バンドマンが楽器を始めてからライブに出るまでの挫折を防ぐためのコミュニティアプリです。
**バンドメンバーのマッチング**と**ライブ出演の促進**を軸に、月額課金で運営します。

## 基本情報

| 項目 | 内容 |
|------|------|
| アプリ名 | BandMatch |
| 技術スタック | Next.js (App Router) + TypeScript + Tailwind CSS v4 |
| 決済 | Stripe（モックUI） |
| バックエンド | なし（モックデータ + localStorage） |
| UIテーマ | ダークモード基調、グラスモーフィズム、グラデーションアクセント |
| カラーパレット | Deep purple/indigo（主色）、Electric blue/cyan（アクセント）、Coral（ハイライト） |

## 主要機能

### 1. ユーザー登録・認証
- メールアドレス + パスワードによるモック認証
- マルチステップ登録フォーム（基本情報 → 音楽プロフィール → 地域・スケジュール）
- パスワード要件: 8文字以上、英字+数字必須
- localStorage ベースのセッション管理
- クロスタブ同期対応

### 2. プロフィール管理
- 基本情報: 名前、ニックネーム、アバター、自己紹介
- 音楽情報: 楽器（複数可）、スキルレベル、ジャンル（複数可）
- 活動情報: 都道府県、市区町村、練習スケジュール、影響を受けたアーティスト
- 表示/編集モード切り替え

### 3. バンドマッチング
**マッチングアルゴリズム（最大155点→100点に正規化）:**

| 項目 | 最大スコア | ロジック |
|------|-----------|---------|
| 地域 | 50pt | 同県35pt、同市50pt |
| 楽器 | 25pt | 補完的=25pt、一部重複=15pt、全重複=5pt |
| ジャンル | 30pt | 共通ジャンル比率 × 30 |
| スキル | 15pt | レベル差が小さいほど高スコア |
| スケジュール | 35pt | 共通曜日比率 × 35 + 時間帯重複ボーナス |

**フィルター:** ジャンル、楽器、地域、スキルレベル

### 4. バンド管理
- バンド作成（名前、説明、ジャンル、地域）
- 楽器スロット制（空きポジション表示）
- メンバー管理（リーダー/メンバー権限）
- 募集中フラグ
- バンド詳細・編集画面

### 5. ライブイベントボード
- イベント一覧（フィルター: 地域、日付、初心者歓迎、ジャンルタグ）
- イベント詳細（会場、日時、料金、登録バンド一覧）
- バンド登録機能
- 初心者歓迎タグ

### 6. コミュニティ
- タイムライン（投稿一覧）
- 投稿タイプ: 一般、練習ログ、マイルストーン、質問
- いいね・コメント機能
- 練習ログ（練習時間記録）
- マイルストーン（初セッション、初ライブ、バンド結成等）

### 7. サブスクリプション管理

| プラン | 月額 | 主な機能 |
|--------|------|----------|
| Free | ¥0 | プロフィール作成、マッチング検索（1日3回）、閲覧のみ |
| Basic | ¥980 | マッチング無制限、バンド作成3つ、メッセージ無制限、イベント作成 |
| Premium | ¥1,980 | 全機能、バンド無制限、優先マッチング、アナリティクス、プレミアムバッジ |

- Stripe モックチェックアウト
- 請求履歴表示

### 8. 管理ダッシュボード
- 統計概要（総ユーザー数、アクティブ数、バンド数、イベント数、月間売上）
- ユーザー管理テーブル（検索、プラン変更、ステータス管理）
- 売上チャート（CSS実装のバーチャート）
- サブスクリプション管理（プラン別内訳）
- アクセス制限: admin権限必須

## セキュリティ対策

| 対策 | 実装箇所 | 説明 |
|------|----------|------|
| 入力バリデーション | `validators.ts` | メール、パスワード、名前等の形式チェック |
| XSS防止 | `security.ts` + React JSX | HTMLエスケープ + Reactの自動エスケープ |
| CSRFトークン | `security.ts` | sessionStorageベースのトークン管理 |
| セキュリティヘッダー | `next.config.ts` | X-Frame-Options, CSP, X-Content-Type-Options |
| 認証ガード | `ProtectedRoute` | 未認証時はログインページへリダイレクト |
| 管理者権限チェック | `ProtectedRoute` (adminOnly) | 非管理者はダッシュボードへリダイレクト |
| localStorage名前空間 | `storage.ts` | `bandmatch-` プレフィックス |

## 検証方法

1. `npm run dev` でローカル起動
2. ユーザー登録 → ログイン → プロフィール編集フロー
3. マッチング検索・フィルター動作
4. バンド作成 → メンバー管理 → イベント登録フロー
5. サブスクリプション選択・モック決済フロー
6. 管理画面のアクセス制御（一般ユーザーでアクセス不可）
7. モバイルレスポンシブ表示
