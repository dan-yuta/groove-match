この案件に初めて参加する方向けの導入ドキュメントです。

プロジェクトの背景から技術スタック、開発の進め方まで、全体像を把握できるようにまとめています。

## 1. このプロジェクトは何か？

### 一言で言うと

**好きなバンドの曲をコピーしたい初心者が「同じ曲をやりたい仲間を見つけてコピバンライブに出る」までをサポートする、クロスプラットフォーム（Web + iOS + Android）アプリ**です。

> **ゴール = コピバンを組んでライブに出ること。**
>
> メインフローは「仲間探し → コピバン結成 → ライブ出演」の一気通貫体験です。
>
> 練習トラッカーやフレンド機能は、そのゴールに到達するまでの挫折を防ぐサポート機能として位置づけています。

### 背景・課題

コピバンをやりたい初心者が直面する問題:

1. **同じ曲をやりたい仲間が見つからない** → 好きなバンドの曲をコピーしたいのに、同じ曲をやりたい人が周りにいない
2. **コピバンの組み方が分からない** → どうやってメンバーを集めればいいか分からない
3. **レベルが合わない** → 経験者ばかりの場所だと萎縮してしまう
4. **ライブに出られない** → コピバンで出られるイベントが見つからない
5. **挫折する** → 孤独に練習して、半年以内にやめてしまう

### ターゲットユーザー

| 項目 | 内容 |
|------|------|
| 年齢 | 18〜35歳 |
| 音楽経験 | 初心者〜中級者（楽器歴0〜3年） |
| 目的 | 好きなバンドの曲をコピーして仲間とライブに出たい |
| 課題 | 同じ曲をやりたい仲間が見つからない / コピバンの組み方が分からない |

### BandMatch が解決すること

**メインフロー（仲間探し → コピバン結成 → ライブ出演）:**

| 課題 | 解決策 |
|------|--------|
| 同じ曲をやりたい仲間がいない | やりたい曲・アーティストで**曲ベースマッチング** |
| コピバンの組み方が分からない | **コピバン作成・セットリスト管理**で簡単にバンド結成 |
| レベルが合わない | スキルレベルを考慮した**スコアリング**で適切な相手を提案 |
| ライブに出られない | **初心者歓迎のコピバンイベント**を掲載、バンドごと登録可能 |

**挫折防止サポート機能:**

| 課題 | 解決策 |
|------|--------|
| 挫折する | **Duolingo型練習トラッカー**（ストリーク・バッジ・ランキング）で継続を支援。<br>毎日の小さな練習を継続する仕組みで挫折を防止 |
| 仲間との繋がりが薄い | **フレンド機能**で練習状況を相互共有。<br>週間ランキングで切磋琢磨 |
| 活動をアピールしにくい | **SNS連携**（X / Instagram / LINE / YouTube）で<br>バンド活動を発信 |

## 6. データの流れ（現在のモック構成）

```
初回アクセス時:
  mockUsers.ts / mockBands.ts / mockEvents.ts / mockPosts.ts
    ↓（初期データとして）
  localStorage に保存
    ↓
  React コンポーネントが読み取り・表示

ユーザー操作時（登録・編集・投稿等）:
  React State を更新
    ↓
  localStorage に書き込み
    ↓
  次回アクセス時も localStorage から復元
```

**重要**: 現在はすべて `localStorage` で動作しています。ブラウザのデータをクリアすると初期状態に戻ります。

### コーディング規約

| 項目 | ルール |
|------|--------|
| 言語 | TypeScript 必須（any 禁止） |
| スタイリング | Tailwind CSS のユーティリティクラスを使用 |
| コンポーネント | 関数コンポーネント + Hooks |
| ファイル命名 | コンポーネント: PascalCase、その他: camelCase |
| インポート | `@/` エイリアスを使用（例: `@/components/ui`） |

## 9. 用語集

| 用語 | 説明 |
|------|------|
| マッチング | ユーザー同士の相性をスコアリングして提案する機能 |
| 楽器スロット | バンドが募集している楽器ポジション（ギター、ドラム等） |
| マイルストーン | 「初セッション」「初ライブ」等の達成イベント |
| 練習ログ | ユーザーが練習時間を記録する機能 |
| グラスモーフィズム | 半透明・ぼかし効果のUIデザイン手法 |
| SSR | Server-Side Rendering。サーバー側でHTMLを生成する方式 |
| App Router | Next.js 13以降のルーティング方式。`app/` ディレクトリ構成 |
| ACU | Aurora Capacity Unit。Aurora Serverlessの処理能力単位 |
| Fargate | AWS ECSのサーバーレスコンテナ実行環境 |
| Capacitor | Web技術（HTML/CSS/JS）で作ったアプリをiOS/Androidネイティブアプリとして配布するためのフレームワーク。WebViewでWebアプリを表示し、ネイティブAPIにアクセスできる。 |

## 10. 困ったときの連絡先
