# BandMatch 起動手順書

このドキュメントでは、BandMatch をローカル環境で起動するまでの手順を説明します。
プログラミング経験が浅い方でも迷わないよう、各ステップを詳しく記載しています。

---

## 1. 前提条件

以下のツールが PC にインストールされている必要があります。

| ツール | 必要バージョン | 確認コマンド | インストール先 |
|--------|--------------|-------------|---------------|
| Node.js | 18.x 以上（推奨: 22.x） | `node -v` | https://nodejs.org/ja （LTS版を選択） |
| npm | 9.x 以上（Node.jsに付属） | `npm -v` | Node.jsと一緒にインストールされます |
| Git | 2.x 以上 | `git --version` | https://git-scm.com/ |

### 確認方法

ターミナル（Windows: PowerShell / Mac: ターミナル）を開いて以下を実行してください。

```bash
node -v
# → v22.22.0 のように表示されればOK

npm -v
# → 10.9.4 のように表示されればOK

git --version
# → git version 2.x.x のように表示されればOK
```

**「command not found」と表示された場合** → 該当ツールがインストールされていません。上記のURLからインストールしてください。

## 3. 依存パッケージのインストール

```bash
npm install
```

- `node_modules/` フォルダが作成され、必要なライブラリがダウンロードされます
- 初回は1〜3分程度かかります
- 「npm warn」が出ても問題ありません（「npm ERR!」が出た場合はエラーです）

### トラブルシューティング

| 症状 | 対処法 |
|------|--------|
| `npm ERR! code EACCES` | `sudo npm install` を試す（Mac/Linux）、または管理者権限でターミナルを開く（Windows） |
| `npm ERR! code ERESOLVE` | `npm install --legacy-peer-deps` を試す |
| とにかくエラーが出る | `rm -rf node_modules package-lock.json` してから再度 `npm install` |

## 4. 開発サーバーの起動

```bash
npm run dev
```

成功すると以下のような出力が表示されます:

```
▲ Next.js 16.1.6 (Turbopack)
  - Local:   http://localhost:3000
  - Network: http://192.168.x.x:3000

 ✓ Starting...
 ✓ Ready in xxxms
```

### ブラウザでアクセス

1. ブラウザを開く
2. アドレスバーに `http://localhost:3000` と入力
3. BandMatch のランディングページが表示されれば成功

### サーバーの停止

ターミナルで `Ctrl + C` を押すとサーバーが停止します。

## 5. テストアカウントでログイン

### 5.1 一般ユーザーとしてログイン

1. トップページの「ログイン」ボタンをクリック
2. 以下の情報を入力:
   - メール: `tanaka@example.com`
   - パスワード: `pass1234`
3. 「ログイン」ボタンをクリック
4. ダッシュボードが表示されれば成功

### 5.2 管理者としてログイン

1. トップページの「ログイン」ボタンをクリック
2. 以下の情報を入力:
   - メール: `admin@bandmatch.jp`
   - パスワード: `admin1234`
3. ダッシュボードからサイドバーの「管理画面」にアクセス可能

### 5.3 新規ユーザーとして登録

1. トップページの「無料で始める」ボタンをクリック
2. 3ステップの登録フォームに入力:
   - Step 1: 名前、メール、パスワード
   - Step 2: 楽器、ジャンル、スキルレベル
   - Step 3: 都道府県、練習スケジュール
3. 登録完了後、自動でダッシュボードへ

## モバイルアプリのビルド

### 前提条件（iOS）
- macOS（Monterey 12以降推奨）
- Xcode 15以降
- CocoaPods（`sudo gem install cocoapods`）
- Apple Developer アカウント

### 前提条件（Android）
- Android Studio Hedgehog以降
- JDK 17
- Android SDK（API Level 33以降）

### 初回セットアップ

```bash
# Capacitor プラットフォーム追加
npx cap add ios
npx cap add android
```

### iOS ビルド

```bash
# ビルド＆同期＆Xcode起動
npm run mobile:ios

# Xcodeで以下を設定:
# 1. Signing & Capabilities → Team を選択
# 2. Bundle Identifier: jp.bandmatch.app
# 3. シミュレータまたは実機を選択して Run
```

### Android ビルド

```bash
# ビルド＆同期＆Android Studio起動
npm run mobile:android

# Android Studioで以下を設定:
# 1. エミュレータまたは実機を選択
# 2. Run ボタンで起動
```

### モバイル開発のトラブルシューティング

| 症状 | 原因 | 対処 |
|------|------|------|
| `cap sync` でエラー | 依存関係不足 | `npm install` を再実行 |
| iOS ビルドエラー | CocoaPods未設定 | `cd ios/App && pod install` |
| Android ビルドエラー | SDK バージョン不一致 | Android Studio で SDK Manager から更新 |
| 画面が白い | ビルド未実行 | `npm run build:mobile` を先に実行 |
| Safe Area が効かない | CSS未適用 | `safe-top` / `safe-bottom` クラスを確認 |

## 9. よくあるトラブルと対処法

### 「ポート3000が使用中」エラー

```bash
# 別のプロセスが3000番ポートを使っている場合

# Mac/Linux: プロセスを探して停止
lsof -i :3000
kill -9 <PID>

# Windows: プロセスを探して停止
netstat -ano | findstr :3000
taskkill /PID <PID> /F

# または別のポートで起動
npm run dev -- -p 3001
```

### ページが真っ白になる

- ブラウザのキャッシュをクリア（Ctrl+Shift+Delete）
- `node_modules` を削除して再インストール:
  ```bash
  rm -rf node_modules .next
  npm install
  npm run dev
  ```

### 「Module not found」エラー

```bash
# パッケージが不足している
npm install

# それでもダメなら
rm -rf node_modules package-lock.json
npm install
```

### ログインできない

- localStorage にデータが保存されています
- ブラウザの開発者ツール → Application → Local Storage → `localhost:3000` を確認
- データをクリアしたい場合: 開発者ツール → Application → Local Storage → Clear All

## 10. 開発時の便利情報

### ホットリロード

- `npm run dev` 中にコードを変更すると、自動でブラウザに反映されます
- ファイルを保存するだけでOK。ブラウザのリロードは不要です

### モックデータの場所

データを変更したい場合は以下のファイルを編集してください:

| ファイル | 内容 |
|----------|------|
| `src/data/mockUsers.ts` | ユーザーデータ（18名） |
| `src/data/mockBands.ts` | バンドデータ（7組） |
| `src/data/mockEvents.ts` | イベントデータ（9件） |
| `src/data/mockPosts.ts` | 投稿データ（18件） |

### ブラウザの開発者ツール

- `F12` キーで開く
- Console タブ: エラーメッセージの確認
- Network タブ: リクエストの確認
- Application タブ: localStorage の確認・編集
