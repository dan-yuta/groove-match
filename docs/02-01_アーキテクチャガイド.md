## ルーティング構造

```
/                    → ランディングページ (公開)
├── /login           → ログイン (auth グループ)
├── /register        → 新規登録 (auth グループ)
├── /dashboard       → ダッシュボード (main グループ, 要認証)
├── /matching        → マッチング (main)
├── /bands           → バンド一覧 (main)
│   ├── /create      → バンド作成
│   └── /[id]        → バンド詳細
├── /events          → イベント一覧 (main)
│   └── /[id]        → イベント詳細
├── /community       → コミュニティ (main)
├── /profile         → プロフィール (main)
├── /subscription    → プラン管理 (main)
└── /admin           → 管理画面 (admin グループ, 要管理者権限)
    ├── /users       → ユーザー管理
    ├── /revenue     → 売上管理
    └── /subscriptions → サブスク管理
```

## コンポーネント依存関係

```
RootLayout
└── ClientProviders
    ├── AuthProvider    ← 認証状態管理
    └── ToastProvider   ← 通知管理
        └── Pages
            ├── UI Components (Button, Input, Card, GlassCard, Badge, Avatar, Modal, Skeleton)
            ├── Layout Components (Header, Footer, MobileNav, ProtectedRoute)
            └── Feature Logic (matching algorithm, storage, validators)
```

## データフロー

```
MockData (初期データ)
    │
    ▼
localStorage (永続化)
    │
    ├── AuthContext.login()  → currentUser セット
    ├── AuthContext.register() → users に追加
    ├── updateUser()        → users 更新
    │
    ▼
React State (UIに反映)
    │
    ▼
Components (描画)
```
