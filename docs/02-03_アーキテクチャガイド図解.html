📚 Next.js App Router 構造

App Router のレイヤー構造を視覚的に表現しています。
外側から内側に向かって、レイアウト → プロバイダー → ルーティング → ページ → コンポーネントと階層化されています。

🏗 RootLayout（ルートレイアウト）
HTML / Body タグ、フォント設定、メタデータ。
全ページに共通するレイアウトを定義。
🔗 ClientProviders（プロバイダー層）
AuthProvider + ToastProvider。
クライアントサイドの状態管理をラップ。
🛤 Route Groups（ルーティング層）
(auth) / (main) / (admin) のルートグループ。
認証・権限によるアクセス制御を実現。
📄 Page Components（ページ層）
各ルートの page.tsx が対応するページコンポーネント。
サーバーコンポーネントとクライアントコンポーネントを併用。
🧩 UI / Feature Components（コンポーネント層）
再利用可能な UI コンポーネントとビジネスロジック。
Button, Card, Modal 等の汎用部品と、matching, storage 等の機能モジュール。

📱 モバイルアプリ構成

Capacitor を使用して、Next.js で構築した Web アプリを iOS / Android ネイティブアプリとしてラップしています。

基本アーキテクチャ

  • 📲 Capacitor がネイティブの WebView 内で Web アプリを表示。
  • 📦 モバイルビルド時は Next.js の静的エクスポート（output: 'export'）を使用。
  • 🔍 src/lib/platform.ts でプラットフォーム（Web / iOS / Android）を検出し、プラットフォーム固有の処理を分岐。

Safe Area 対応

  • 📏 ノッチ付きデバイス（iPhone X 以降等）に対応するため、Safe Area Insets を CSS で適用しています。
