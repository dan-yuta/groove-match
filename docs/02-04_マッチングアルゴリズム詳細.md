# マッチングアルゴリズム詳細

## スコア算出ロジック

### 総合スコア = 地域 + 楽器 + ジャンル + スキル + スケジュール + 曲マッチング (最大160pt → 100に正規化)

コピバン特化のマッチングとして、**やりたい曲の一致**を最重要スコア要素として導入しています。

---

### 1. 地域スコア (最大40pt)

| 条件 | スコア |
|------|--------|
| 同じ都道府県 + 同じ市区町村 | 40pt |
| 同じ都道府県のみ | 25pt |
| 異なる都道府県 | 0pt |

### 2. 楽器スコア (最大20pt)

補完的な楽器構成を高く評価（コピバン結成に有利）

| 条件 | スコア |
|------|--------|
| 完全に異なる楽器 | 20pt |
| 一部重複 | 12pt |
| 全て同じ楽器 | 4pt |

### 3. ジャンルスコア (最大20pt)

```
スコア = (共通ジャンル数 / 自分のジャンル数) × 20
```

### 4. スキルスコア (最大15pt)

近いスキルレベル同士を高く評価

```
レベル差 = |自分のスキル値 - 相手のスキル値|
スコア = max(0, 15 - レベル差 × 5)

スキル値: beginner=1, intermediate=2, advanced=3, expert=4
```

| レベル差 | スコア |
|---------|--------|
| 0 (同じ) | 15pt |
| 1 | 10pt |
| 2 | 5pt |
| 3 | 0pt |

### 5. スケジュールスコア (最大30pt)

```
基本スコア = (共通曜日数 / 自分のスケジュール数) × 30
ボーナス = 共通曜日で時間帯が重複している場合 +5pt/曜日
```

### 6. 曲マッチングスコア (最大35pt) ★コピバン特化

コピバンの核心である「同じ曲をやりたい」マッチングを最重要スコアとして算出します。

#### 6.1 やりたい曲の一致 (最大35pt)

```
基本スコア = 一致曲数 × 7（最大5曲 = 35pt）
```

| 一致曲数 | スコア |
|---------|--------|
| 5曲以上 | 35pt（上限） |
| 4曲 | 28pt |
| 3曲 | 21pt |
| 2曲 | 14pt |
| 1曲 | 7pt |
| 0曲 | 0pt（アーティストボーナスで加点の可能性あり） |

#### 6.2 好きなアーティスト一致ボーナス (曲スコア35ptの内数)

やりたい曲が完全一致しなくても、好きなアーティストが同じなら親和性が高いと判断。

```
アーティストボーナス = 一致アーティスト数 × 3（最大15pt、ただし曲スコアとの合計で35ptが上限）
```

#### 6.3 同アーティスト別曲ボーナス

同じアーティストの違う曲をやりたい場合も、コピバンとしての親和性が高い。

```
同アーティスト別曲ボーナス = 該当数 × 2（最大10pt、曲スコア合計35ptの内数）
```

#### 計算例

**ユーザーA:**
- やりたい曲: 天体観測(BUMP)、花の名(BUMP)、完全感覚Dreamer(ONE OK ROCK)
- 好きなアーティスト: BUMP OF CHICKEN、ONE OK ROCK

**ユーザーB:**
- やりたい曲: 天体観測(BUMP)、ロストワンの号哭(Neru)、完全感覚Dreamer(ONE OK ROCK)
- 好きなアーティスト: BUMP OF CHICKEN、Neru

**曲スコア計算:**
- 曲一致: 天体観測 + 完全感覚Dreamer = 2曲 → 14pt
- アーティスト一致: BUMP OF CHICKEN = 1 → +3pt
- 同アーティスト別曲（Aの花の名 と Bの天体観測は同BUMP）→ すでに曲一致でカウント済み
- **合計: 17pt / 35pt**

---

## 補足: マッチングに活用可能な追加データ

マッチングアルゴリズム本体は上記6要素で算出しますが、以下のユーザーデータも将来的なマッチング精度向上に活用可能です。

- **練習データ（PracticeLog）**: ユーザーの練習頻度・練習曲の傾向から、アクティブ度や曲の習熟度を推定可能
- **フレンド関係（FriendRelation）**: 共通のフレンドが多いユーザー同士は親和性が高い可能性がある（ソーシャルグラフ活用）
- **ストリーク・バッジ**: 継続的に練習しているユーザー同士のマッチングを優先するオプションの検討
