ルーティング構造

src/app/ ディレクトリのフォルダ構成がURLパスに対応。括弧はルートグループ。

URL構成

/                    → ランディングページ（公開）
|
|-- (auth)             → 認証グループ（URLに含まれない）
|   |-- /login         → ログイン
|   |-- /register      → 新規登録（Pro + CC登録必須）
|
|-- (main)             → メイングループ（要認証）
|   |-- /dashboard     → ダッシュボード
|   |-- /matching      → マッチング検索
|   |-- /bands         → コピバン一覧・詳細
|   |-- /events        → ライブイベントボード
|   |-- /community     → コミュニティ（タイムライン）
|   |-- /profile       → プロフィール
|   |-- /subscription  → サブスクリプション管理
|
|-- (admin)            → 管理者グループ（admin権限必須）
    |-- /admin         → 管理ダッシュボード
    |-- /admin/users   → ユーザー管理
    |-- /admin/sales   → 売上管理

🔌 API Routes / Server Actions

Next.jsに組み込まれたサーバーサイド機能。別途バックエンドサーバー不要。

🗃 Prisma ORM

データベースをTypeScriptコードで操作。SQLを直接書かなくてよい。

🗃 RDS PostgreSQL

AWSマネージドDB。初期は db.t4g.micro。成長期にAurora移行を検討。

🔒 Amazon Cognito

認証サービス。ソーシャルログイン（Google/X/Instagram）対応。5万MAUまで無料。

パスワード8文字以上、英字+数字
JWT有効期限24h / リフレッシュ30日

💰 Stripe

Web決済処理。手数料3.6%。iOS/Androidはアプリ内課金と併用。

⚡ ElastiCache (Redis)

Spotify APIレスポンスキャッシュ + セッション管理。API呼び出し回数を削減。

データベース設計（主要テーブル）

RDS PostgreSQL + Prisma。曲データはSpotify IDのみ保存し、表示時にAPIから取得。

主要テーブル

User

  • id, email, name
  • nickname, avatar
  • instruments[], genres[]
  • skillLevel, prefecture
  • wantToPlaySongs[]
  • canPlaySongs[]
  • role (user/admin)

Band

  • id, name, description
  • genre, prefecture
  • targetArtist
  • setlist[] (Spotify ID)
  • members[], leaderId
  • isRecruiting

Event

  • id, title, venue
  • date, fee
  • prefecture, genreTags[]
  • isBeginnerFriendly
  • registeredBands[]
  • createdBy, isOfficial

Subscription

  • id, userId
  • plan (free/pro)
  • stripeCustomerId
  • trialEndsAt
  • inviteCode

PracticeLog

  • id, userId, songId
  • duration, date
  • streak, totalMinutes

FriendRelation

  • id, requesterId
  • addresseeId
  • status (pending/accepted)
