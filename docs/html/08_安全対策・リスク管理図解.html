運営の責任範囲

Groove Matchは「音楽活動のマッチングプラットフォーム」です。ユーザーを繋ぐ「場」を提供し、安全ツールを整備します。

✅ 運営が責任を負う

  • ✓ 安全なツールの提供（通報・ブロック機能）
  • ✓ 通報への合理的な期間内での対応
  • ✓ 禁止行為の明示と違反者への対応
  • ✓ 安全な利用のための注意喚起
  • ✓ プライバシーポリシーに基づくデータ管理

⚠ ユーザーの自己責任

  • ✗ ユーザー間の金銭トラブル
  • ✗ 外部ツール（LINE等）でのやり取り
  • ✗ ライブ出演に関するトラブル（キャンセル等）
  • ✗ ユーザーが提供した情報の真偽
  • ✗ ユーザーが自発的に公開した情報の悪用

⚠ 免責条項の限界

免責条項は「重大な過失」を免除しません。例えば、犯罪行為の通報を放置した場合は過失に該当する可能性があります。
通報には必ず合理的な期間内に対応し、対応記録を残す必要があります。

通報・エスカレーションフロー

通報は匿名で処理されます。通報者の情報は対象ユーザーに通知されません。

1. ユーザーが通報

通報ボタン → カテゴリ選択 → 詳細入力（任意）

▼

2. 通報キューに追加

カテゴリ別に優先度を自動判定

▼

3. 運営がレビュー

事実確認 → 過去の通報履歴 → メッセージ履歴確認

▼

4. 対応を実行

警告 / 機能制限 / アカウント停止 / 永久BAN

▼

5. 通報者に結果通知

「対応しました」のみ通知。具体的な制裁内容は非公開

コミュニケーション設計

初回接触はアプリ内で安全に。信頼構築後は外部ツールへ自然に移行する3フェーズモデル。

Phase 1

アプリ内メッセージ

初回接触は安全なアプリ内で

  • ✓ Pro限定機能
  • ✓ 運営が監視可能
  • ✓ 不審パターン検出
  • ✓ 課金転換の動機
→
Phase 2

外部連絡先の交換

信頼構築後、ユーザーの判断で

  • ⚠ 注意喚起を1回表示
  • ⚠ 自己責任の明示
  • — 運営は関与しない
→
Phase 3

外部ツールでバンド活動

LINEグループ等で実務連絡

  • — 練習日時の調整
  • — グループ通話
  • — ファイル共有
  • — 完全に運営の責任範囲外

💬 なぜハイブリッド方式なのか？

アプリ内のみだと、バンド活動の実務（グループ通話、日程調整等）にはLINEの方が圧倒的に便利で、ユーザー体験が悪化します。
外部のみだと、初対面の相手にいきなりLINEを教えるのは不安で、初回接触時の安全網がなくなります。
ハイブリッド方式なら、初回はアプリ内で安全に接触し、信頼関係ができたら自然にLINEへ移行できます。
