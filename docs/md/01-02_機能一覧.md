# Groove Match - 機能一覧

> 機能の優先順位: **1. マッチング（仲間探し）→ 2. コピバン結成（セットリスト管理）→ 3. ライブ出演（イベントボード）** がメインフロー。
> 練習トラッカー・フレンド・ランキングは「挫折防止のサポート機能」です。

アプリ全体像については [01-01_アプリ概要.md](./01-01_アプリ概要.md) を参照してください。

---

## 1. ユーザー登録・認証

- **ソーシャルログイン**: Google / X (Twitter) / Instagram アカウントでワンクリック登録・ログイン
- メールアドレス + パスワードによる認証（Amazon Cognito = AWSの認証サービス）
- マルチステップ登録フォーム（基本情報 → 音楽プロフィール → コピバン設定 → 地域・スケジュール → SNS連携）
- **初回登録時にProプラン選択 + クレジットカード登録が必須**
- 招待コード入力欄（任意）: 入力すると**トライアル期間が2週間→1ヶ月に延長**
- パスワード要件: 8文字以上、英字+数字必須
- JWT（JSON Web Token = サーバーがユーザーを識別するための暗号化された文字列）ベースのセッション管理
- クロスタブ同期対応

---

## 2. プロフィール管理

- 基本情報: 名前、ニックネーム、アバター、自己紹介
- **年代表示**: 生年月日から「20代前半」「30代後半」のように表示（生年月日は非公開）
- 音楽情報: 楽器（複数可）、スキルレベル、ジャンル（複数可）
- コピバン情報: コピーしたいアーティスト、やりたい曲（wantToPlaySongs）、弾ける曲（canPlaySongs）
  - **曲の登録はSpotify API（音楽ストリーミングサービスのAPI）で検索し、IDだけをDBに保存**
  - 表示時にSpotify APIから最新のアーティスト名・曲名・ジャケット画像を取得
- 活動情報: 都道府県、市区町村、練習スケジュール、影響を受けたアーティスト
- 表示/編集モード切り替え

---

## 3. コピバンマッチング

6要素でスコアリングし、相性の良いユーザーを自動で提案します。

| 項目 | 最大スコア | ロジック |
|------|-----------|---------|
| 地域 | 40pt | 同県25pt、同市40pt |
| 楽器 | 20pt | 補完的=20pt、一部重複=12pt、全重複=4pt |
| ジャンル | 20pt | 共通ジャンル比率 x 20 |
| スキル | 15pt | レベル差が小さいほど高スコア |
| スケジュール | 30pt | 共通曜日比率 x 30 + 時間帯重複ボーナス |
| 曲マッチング | 35pt | やりたい曲+弾ける曲の一致数 x 7（最大5曲分）+ 好きなアーティスト一致ボーナス |

合計最大160pt → **100pt上限でクランプ**（100点を超えた場合は100点に丸める）。

詳細は [02-01_マッチングアルゴリズム.md](./02-01_マッチングアルゴリズム.md) を参照してください。

**フィルター:** やりたい曲、アーティスト、ジャンル、楽器、地域、スキルレベル

---

## 4. コピバン管理

- コピバン作成（名前、説明、コピー対象アーティスト/曲、ジャンル、地域）
- **セットリスト管理**: コピーする曲の一覧・順序管理（Spotify APIで曲情報を取得）
- 楽器スロット制（空きポジション表示）: コピバンに「何が足りないか」が一目瞭然
- メンバー管理（リーダー/メンバー権限）
- 募集中フラグ
- コピバン詳細・編集画面

---

## 5. ライブイベントボード

- イベント一覧（フィルター: 地域、日付、初心者歓迎、ジャンルタグ）
- イベント詳細（会場、日時、料金、登録バンド一覧）
- バンド登録機能
- 初心者歓迎タグ
- **イベント登録: 運営 + Proユーザーの両方が可能**
  - 運営が登録したイベントには**「公式」バッジ**が表示される

---

## 6. コミュニティ

- タイムライン（投稿一覧）
- 投稿タイプ: 一般、練習ログ、マイルストーン（初セッション・初ライブ等の達成記録）、質問、カバー動画
- いいね・コメント機能
- 練習ログ（練習時間記録）
- マイルストーン表示

---

## 7. 挫折防止サポート機能（Duolingo型ゲーミフィケーション）

> メインゴール「コピバンを組んでライブに出る」に到達するまでの挫折を防ぐサポート機能群です。

- **練習トラッカー** - 曲ごとの練習時間記録、ストリーク（連続練習日数）、週間目標
- **フレンド機能** - **相互承認型**（フォロー/フォロワー形式ではなく、双方が承認して初めてフレンドになる）
  - 練習状況の相互共有、応援機能
  - **練習ランキングは相互フレンド間で表示**
- **バッジ＆マイルストーン** - 12種類のバッジ（ストリーク、累計時間、初ライブ等）
- **通知システム** - 練習リマインド、ストリーク危機、フレンド活動、ランキング変動
- **ソーシャルログイン** - Google / X / Instagram でのアカウント作成・ログイン
- **SNS連携** - X / Instagram / LINE / YouTube アカウント連携（プロフィール表示用）

## 8. サブスクリプション・課金モデル

### プラン比較

| 機能 | Free (¥0) | Pro (¥500/月) |
|------|-----------|---------------|
| プロフィール作成 | ○ | ○ |
| マッチング検索 | 1日3回 | **無制限** |
| コミュニティ閲覧 | ○ | ○ |
| イベント閲覧 | ○ | ○ |
| バンド作成 | 1つまで | **無制限** |
| メッセージ | - | **○** |
| イベント作成 | - | **○（「公式」バッジなし）** |
| アナリティクス | - | **○** |
| フレンド | 5人まで | **無制限** |
| ランキング | フレンドのみ | **全体ランキング** |
| 通知カスタマイズ | 基本のみ | **全カスタマイズ** |
| SNS連携 | 1つ | **無制限** |

### 課金フロー

1. **初回登録時**: Proプランが自動選択され、**クレジットカード登録が必須**
2. **トライアル期間**: 2週間無料（招待コード入力で**1ヶ月に延長**）
3. **トライアル終了後**: 解約しなければ**自動で¥500/月の課金が開始**
4. **解約**: いつでも可能。解約するとFreeプランにダウングレード
5. **自動更新**: Freeプランにダウングレードしない限り毎月自動更新

### 法的注意事項（必須表示）

- 登録時に「トライアル後は自動でPro ¥500/月に移行する」旨を**明確に表示**
- **解約方法を分かりやすく案内**する
- **トライアル終了3日前にメール通知**を送信

### 招待コード機能

既存ユーザーが新規ユーザーを招待できる仕組みです。

| ステップ | 内容 |
|---------|------|
| 1 | 既存ユーザーがアプリ内で**招待コードを生成** |
| 2 | 新規ユーザーが登録時に招待コードを入力 |
| 3 | 新規ユーザー: トライアル期間が**2週間 → 30日に延長** |
| 4 | 紹介者: **Proプラン1週間延長**の特典 |

### 決済戦略（プラットフォーム別）

| 決済経路 | 手数料 | ¥500の手取り | 実装 |
|---------|--------|-------------|------|
| Web（Stripe） | 3.6% | ¥482 | Stripe Checkout |
| iOS（App Store） | 15%※ | ¥425 | StoreKit 2 |
| Android（Google Play） | 15%※ | ¥425 | Google Play Billing |

> ※ Apple Small Business Program / Google Play サブスク初年度料率を適用（年間売上$1M以下）

**併用戦略**: Web経由はStripe決済（手取り最大）、iOS/AndroidはApp内課金（UXシームレス）を併用します。
加重平均手取りは **¥449/人** となります。

詳細なコスト計算は [04-02_コスト計算.md](./04-02_コスト計算.md) を参照してください。

## 10. アーティスト・曲データの仕組み

Groove Matchでは、アーティストや曲のデータを自前のDBに保持せず、**外部APIから取得**します。

| API | 用途 |
|-----|------|
| **Spotify API** | 曲の検索・メタ情報（曲名、アーティスト名、ジャケット画像）取得 |
| **MusicBrainz** | オープンソースの音楽データベース。Spotify APIの補完として使用 |

### データフロー

1. ユーザーが曲を登録するとき → **Spotify APIで検索**し、曲のIDだけをDBに保存
2. 画面に表示するとき → **Spotify APIから最新情報を取得**して表示
3. **キャッシュはRedis**（インメモリデータベース = メモリ上にデータを保持する高速なDB）に保存し、APIの呼び出し回数を削減

この方式により、常に最新のアーティスト・曲情報を表示でき、DB容量も節約できます。

詳細は [03-02_バックエンド設計.md](./03-02_バックエンド設計.md) を参照してください。
