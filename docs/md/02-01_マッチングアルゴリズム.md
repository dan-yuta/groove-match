# Groove Match - マッチングアルゴリズム

## スコア算出ロジック

### 総合スコア = 地域 + 楽器 + ジャンル + スキル + スケジュール + 曲マッチング (最大160pt → 100pt上限でクランプ)

「クランプ」とは、計算結果が100点を超えた場合に100点に丸める処理のことです。

コピバン特化のマッチングとして、**やりたい曲の一致**を最重要スコア要素として導入しています。

曲データは**Spotify API**（音楽ストリーミングサービスの検索API）で検索し、IDだけをDBに保存します。マッチング時はこのIDを比較して一致判定を行います。

---

### 1. 地域スコア (最大40pt)

練習やライブは対面で行うため、地域の近さを高く評価します。

| 条件 | スコア |
|------|--------|
| 同じ都道府県 + 同じ市区町村 | 40pt |
| 同じ都道府県のみ | 25pt |
| 異なる都道府県 | 0pt |

### 2. 楽器スコア (最大20pt)

コピバンを組むには、異なる楽器（ギター・ベース・ドラムなど）が揃っている方が有利です。そのため、**補完的な楽器構成**（お互いが違う楽器を持っている状態）を高く評価します。

| 条件 | スコア |
|------|--------|
| 完全に異なる楽器（補完的） | 20pt |
| 一部重複 | 12pt |
| 全て同じ楽器 | 4pt |

### 3. ジャンルスコア (最大20pt)

好きなジャンルの共通度合いを評価します。

```
スコア = (共通ジャンル数 / 自分のジャンル数) x 20
```

例: 自分がロック・ポップ・アニソンの3ジャンル、相手がロック・アニソンの場合
→ 共通2 / 自分3 x 20 = 13.3pt

### 4. スキルスコア (最大15pt)

スキルレベルが近い人同士を高く評価します。初心者が上級者とマッチングして萎縮してしまうことを防ぎます。

```
レベル差 = |自分のスキル値 - 相手のスキル値|
スコア = max(0, 15 - レベル差 x 5)
```

スキル値の定義:
- beginner（初心者）= 1
- intermediate（中級者）= 2
- advanced（上級者）= 3
- expert（エキスパート）= 4

| レベル差 | スコア | 例 |
|---------|--------|-----|
| 0（同じ） | 15pt | 初心者 vs 初心者 |
| 1 | 10pt | 初心者 vs 中級者 |
| 2 | 5pt | 初心者 vs 上級者 |
| 3 | 0pt | 初心者 vs エキスパート |

### 5. スケジュールスコア (最大30pt)

練習可能な曜日・時間帯の一致度を評価します。

```
基本スコア = (共通曜日数 / 自分のスケジュール数) x 30
ボーナス = 共通曜日で時間帯が重複している場合 +5pt/曜日
```

例: 自分が月・水・土の3日、相手が水・土の場合
→ 共通2 / 自分3 x 30 = 20pt（+ 時間帯が重複していれば追加ボーナス）

### 6. 曲マッチングスコア (最大35pt) ★コピバン特化

コピバンの核心である「同じ曲をやりたい」マッチングを最重要スコアとして算出します。

#### 6.1 曲の一致（やりたい曲 + 弾ける曲）(最大35pt)

```
allCurrentSongs = wantToPlaySongs + canPlaySongs  （自分の全曲）
allTargetSongs = 相手のwantToPlaySongs + canPlaySongs  （相手の全曲）
基本スコア = 共通曲数 x 7（最大5曲 = 35pt）
```

| 一致曲数 | スコア |
|---------|--------|
| 5曲以上 | 35pt（上限） |
| 4曲 | 28pt |
| 3曲 | 21pt |
| 2曲 | 14pt |
| 1曲 | 7pt |
| 0曲 | 0pt（アーティストボーナスで加点の可能性あり） |

#### 6.2 好きなアーティスト一致ボーナス (曲スコア35ptの内数)

やりたい曲が完全一致しなくても、好きなアーティストが同じなら親和性が高いと判断します。

```
アーティストボーナス = 一致アーティスト数 x 5（ただし曲スコアとの合計で35ptが上限）
```

> ※ 同アーティスト別曲ボーナスは現在のコード実装には含まれていません。将来的な拡張候補です。

## 計算例

**ユーザーA:**
- やりたい曲: 天体観測(BUMP)、花の名(BUMP)、完全感覚Dreamer(ONE OK ROCK)
- 好きなアーティスト: BUMP OF CHICKEN、ONE OK ROCK

**ユーザーB:**
- やりたい曲: 天体観測(BUMP)、ロストワンの号哭(Neru)、完全感覚Dreamer(ONE OK ROCK)
- 好きなアーティスト: BUMP OF CHICKEN、Neru

**曲スコア計算:**
- 曲一致: 天体観測 + 完全感覚Dreamer = 2曲 → 14pt
- アーティスト一致: BUMP OF CHICKEN = 1 → +5pt
- **合計: 19pt / 35pt**

## 補足: マッチングに活用可能な追加データ

マッチングアルゴリズム本体は上記6要素で算出しますが、以下のユーザーデータも将来的なマッチング精度向上に活用可能です。

- **練習データ（PracticeLog）**: ユーザーの練習頻度・練習曲の傾向から、アクティブ度や曲の習熟度を推定可能
- **フレンド関係（FriendRelation）**: 共通のフレンドが多いユーザー同士は親和性が高い可能性がある（ソーシャルグラフ活用）
- **ストリーク・バッジ**: 継続的に練習しているユーザー同士のマッチングを優先するオプションの検討
