## 技術構成図（クロスプラットフォーム）

```
[iOS App Store] [Google Play] [Web Browser]
        |              |           |
    Capacitor      Capacitor    直接アクセス
        |              |           |
        +------+-------+-----------+
               |
      Next.js App Router
```

Capacitor（キャパシター）は、Webアプリを iOS/Android のネイティブアプリとして配布するためのフレームワークです。WebView（アプリ内ブラウザ）でWebアプリを表示し、カメラやプッシュ通知などのネイティブ機能にもアクセスできます。

## ルーティング構造

Next.js App Routerでは、`src/app/` ディレクトリのフォルダ構成がそのままURLのパスになります。`(auth)` のように括弧で囲んだフォルダは「ルートグループ」と呼ばれ、URLには含まれませんがレイアウトを共有できます。

```
/                    → ランディングページ（公開）
|-- /login           → ログイン (auth グループ)
|-- /register        → 新規登録 (auth グループ)
|-- /dashboard       → ダッシュボード (main グループ, 要認証)
|-- /matching        → マッチング (main)
|-- /bands           → バンド一覧 (main)
|   |-- /create      → バンド作成
|   +-- /[id]        → バンド詳細（[id]は動的パラメータ）
|-- /events          → イベント一覧 (main)
|   +-- /[id]        → イベント詳細
|-- /community       → コミュニティ (main)
|-- /practice        → 練習トラッカー (main)
|-- /ranking         → ランキング (main)
|-- /profile         → プロフィール (main)
|-- /subscription    → プラン管理 (main)
+-- /admin           → 管理画面 (admin グループ, 要管理者権限)
    |-- /users       → ユーザー管理
    |-- /revenue     → 売上管理
    +-- /subscriptions → サブスク管理
```

## コンポーネント依存関係

「コンポーネント」とは、画面を構成する再利用可能な部品のことです。ボタン、カード、入力欄などの小さな部品を組み合わせて画面を構築します。

```
RootLayout（ルートレイアウト = 全ページ共通の枠組み）
+-- ClientProviders
    |-- AuthProvider    ← 認証状態管理（ログイン中かどうか）
    +-- ToastProvider   ← 通知管理（操作完了メッセージ等）
        +-- Pages（各ページ）
            |-- UI Components (Button, Input, Card, GlassCard, Badge, Avatar, Modal)
            |-- Layout Components (Header, Footer, MobileNav, ProtectedRoute)
            +-- Feature Logic (matching algorithm, storage, validators)
```

### UI コンポーネント一覧

| コンポーネント | 用途 |
|--------------|------|
| Button | ボタン（variant: primary/secondary/ghost等） |
| Input | 入力フィールド |
| Card | カード（情報をまとめて表示する枠） |
| GlassCard | グラスモーフィズムカード（半透明・ぼかし効果付き） |
| Avatar | アバター（ユーザーのプロフィール画像、オンライン表示付き） |
| Badge | バッジ（タグやラベルの表示） |
| Modal | モーダル（画面の上に重ねて表示するダイアログ） |
| Toast | トースト通知（画面の端に一時的に表示するメッセージ） |

### レイアウトコンポーネント

| コンポーネント | 用途 |
|--------------|------|
| Header | ヘッダー（デスクトップナビゲーション・通知ベル） |
| Footer | フッター |
| MobileNav | モバイルタブバー（画面下部の5タブナビゲーション） |
| ProtectedRoute | 認証ガード（未ログイン時はログインページへリダイレクト） |

## データフロー（現在のモック構成）

```
初回アクセス時:
  mockUsers.ts / mockBands.ts / mockEvents.ts / mockPosts.ts
    |（初期データとして）
    v
  localStorage に保存
    |
    v
  React コンポーネントが読み取り・表示

ユーザー操作時（登録・編集・投稿等）:
  React State を更新
    |
    v
  localStorage に書き込み
    |
    v
  次回アクセス時も localStorage から復元
```

**重要**: 現在はすべて `localStorage`（ブラウザにデータを保存する仕組み）で動作しています。ブラウザのデータをクリアすると初期状態に戻ります。

## モバイルアプリ構成

### 基本アーキテクチャ

- **Capacitor** がネイティブの WebView 内で Web アプリを表示
- モバイルビルド時は Next.js の静的エクスポート（`output: 'export'`）を使用
- Capacitor のAPIでプラットフォーム（Web / iOS / Android）を検出し、プラットフォーム固有の処理を分岐

### Safe Area 対応

ノッチ付きデバイス（iPhone X以降等）に対応するため、Safe Area Insets（画面の安全な表示領域）を CSS で適用しています。

### ネイティブプラグイン

| プラグイン | 用途 |
|-----------|------|
| StatusBar | ステータスバー（画面上部の時計・電波表示エリア）のスタイル制御 |
| SplashScreen | スプラッシュスクリーン（アプリ起動時の表示画面）制御 |
| PushNotifications | プッシュ通知の受信・処理 |
| Haptics | 触覚フィードバック（タップ・操作時の振動） |
| Keyboard | キーボード表示/非表示のハンドリング |
