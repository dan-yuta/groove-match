# Groove Match - バックエンド設計

## 概要

Groove Matchのバックエンドは、Next.js API Routes / Server Actions をAPIレイヤーとし、RDS PostgreSQL をメインDB、Amazon Cognito で認証、Spotify API / MusicBrainz で曲データを取得する構成です。

> **現在の状態**: フロントエンドはモックデータ + localStorageで動作しています。本ドキュメントは、バックエンド構築時の設計方針です。

---

## 技術スタック

| 技術 | 用途 | 補足説明 |
|------|------|---------|
| Next.js API Routes / Server Actions | APIエンドポイント | Next.jsに組み込まれたサーバーサイド機能。別途バックエンドサーバーを立てる必要がない |
| Prisma | ORM（DB操作） | データベースをTypeScriptのコードで操作できるツール。SQLを直接書かなくてよい |
| RDS PostgreSQL | メインDB | AWSのマネージドデータベースサービス。初期は db.t4g.micro で開始 |
| Amazon Cognito | ユーザー認証 | AWSの認証サービス。ソーシャルログイン（Google/X/Instagram）にも対応 |
| Stripe | 決済処理 | クレジットカード決済を安全に処理するサービス |
| ElastiCache (Redis) | キャッシュ | メモリ上にデータを保持する高速なDB。Spotify APIのレスポンスキャッシュ等に使用 |
| API Gateway WebSocket | リアルタイムチャット | WebSocketプロトコルによるリアルタイム双方向通信。チャット機能の基盤 |
| Lambda | WebSocketハンドラー | WebSocket接続管理、メッセージ送受信、チャットルーム管理をサーバーレスで処理 |

## 認証設計 (Amazon Cognito)

Amazon Cognito（コグニート）は、AWSが提供するユーザー認証サービスです。パスワードの安全な保存、ソーシャルログイン、JWTトークンの発行などを自動で行ってくれます。

### 認証フロー

```
ユーザー → Cognito → JWT トークン発行
                         |
                         v
              API Routes でトークン検証
                         |
                         v
              DB操作（Prisma経由）
```

### 設定項目

| 項目 | 設定 |
|------|------|
| ユーザープール | groovematch-users |
| パスワードポリシー | 8文字以上、英字+数字必須 |
| MFA（多要素認証） | 任意（将来対応） |
| ソーシャルログイン | Google / X / Instagram（Apple連携は将来対応予定） |
| JWTトークン有効期限 | アクセストークン: 24時間、リフレッシュトークン: 30日 |
| 5万MAUまで | 無料 |

## データベース設計 (RDS PostgreSQL)

### DB選定理由

| 項目 | 説明 |
|------|------|
| **初期**: RDS PostgreSQL (db.t4g.micro) | 初期フェーズでAurora Serverless v2は過剰。RDS PostgreSQLで**月約¥10,000のコスト削減**が可能 |
| **移行タイミング**: ユーザー数5,000超 | ユーザー数が5,000を超えたらAurora PostgreSQLに移行 |
| **移行コスト**: 最小限 | AuroraもPostgreSQLエンジンなので、Prisma含めて移行コストは最小限。スナップショットからの復元で移行可能 |

### ORM: Prisma

Prisma（プリズマ）は、TypeScriptでデータベースを操作するためのORM（Object-Relational Mapping）ツールです。SQLを直接書く代わりに、TypeScriptのコードでデータの取得・作成・更新・削除ができます。

```typescript
// 例: ユーザーを取得するPrismaのコード
const user = await prisma.user.findUnique({
  where: { email: "tanaka@example.com" },
  include: { instruments: true, genres: true }
});
```

## 外部API連携 (Spotify API / MusicBrainz)

### Spotify API

Spotify API は、Spotifyの楽曲データベースを検索・取得できるAPIです。

| 用途 | エンドポイント | 説明 |
|------|---------------|------|
| 曲検索 | GET /v1/search?type=track | ユーザーが曲を登録するときに検索 |
| 曲情報取得 | GET /v1/tracks/{id} | 曲名・アーティスト名・ジャケット画像の取得 |
| アーティスト情報 | GET /v1/artists/{id} | アーティストのメタ情報取得 |

### MusicBrainz

MusicBrainz は、オープンソースの音楽データベースです。Spotify APIの補完として、Spotifyに登録されていない曲のデータ取得に使用します。

### データフロー

```
ユーザーが曲を登録:
  1. フロントエンドで曲名を入力
  2. Spotify API で検索（リアルタイムサジェスト）
  3. ユーザーが曲を選択
  4. Spotify の track ID だけを DB に保存
     （曲名・アーティスト名などはDBに保存しない）

画面に曲を表示:
  1. DB から Spotify track ID を取得
  2. Redis キャッシュを確認
     - キャッシュあり → キャッシュから返す
     - キャッシュなし → Spotify API から取得 → Redis に保存
  3. 曲名・アーティスト名・ジャケット画像を表示
```

### キャッシュ戦略 (Redis)

| データ | TTL（有効期限） | 理由 |
|--------|---------------|------|
| 曲情報 | 24時間 | 曲のメタ情報は頻繁に変わらない |
| アーティスト情報 | 24時間 | 同上 |
| 検索結果 | 1時間 | 検索結果は新しい曲が追加される可能性がある |
| マッチング結果 | 30分 | ユーザーの設定変更を反映するため短めに設定 |

## セキュリティ対策

| 対策 | 実装箇所 | 説明 |
|------|----------|------|
| 入力バリデーション | API Routes | サーバーサイドで全入力を検証（不正なデータを弾く） |
| XSS防止 | React JSX | Reactの自動エスケープ（HTMLタグを無害化する機能） |
| CSRFトークン | next.config.ts | CSRF（クロスサイトリクエストフォージェリ = 偽のリクエストを送る攻撃）を防ぐトークン |
| セキュリティヘッダー | next.config.ts | X-Frame-Options, CSP（コンテンツセキュリティポリシー）, X-Content-Type-Options |
| 認証ガード | ProtectedRoute + API Middleware | 未認証ユーザーのアクセスをブロック |
| 管理者権限チェック | API Middleware | 非管理者からの管理APIアクセスをブロック |
| 通信暗号化 | ALB + CloudFront | TLS 1.2以上（HTTPS必須） |
| パスワード保存 | Cognito | bcrypt（パスワードを暗号化するアルゴリズム）で保存 |
| SQLインジェクション対策 | Prisma | パラメータバインディング（ORMが自動で安全なクエリを生成） |
| WAF | CloudFront | AWS WAF（Web Application Firewall）によるボット対策・レートリミット |
| シークレット管理 | Secrets Manager | DB接続情報・APIキーを暗号化して管理。自動ローテーション有効 |

  city: string;
  instruments: UserInstrument[];
  genres: string[];
  schedule: Schedule[];
